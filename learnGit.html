1.git add Text.html //添加文件到仓库
2.git commit -m "我提交了Text.html文件" //提交文件到仓库
（在这里我们修改了Text.html文件）
3.git status //查看git当前状态 (这里会提示我修改了，但没提交)
（如果我们出去了很久，回来时忘记修改了那些年了，我们可以用diff）
4.git diff text.html
版本控制系统肯定有某个命令可以告诉我们历史记录，在Git中，我们用git log命令查看：
5.git log (可选参数 --pretty=oneline) 一大堆是乱码是版本号
（git中 HEAD 表示当前版本 HEAD^上一个版本）
6. git reset --hard HEAD^ (回退到上一版本)
（这时当你用 git log时 你发现 最新版本的那个已经没了，退不回去了怎么办！？）
当你的命令行窗口还没有关掉时，你可以继续往上翻，找到未来的那个最新的版本号然后
7. git reset --hard 1094a24fgt1fft4t5
(如果回退版本后 你关闭了命令行窗口，并且关闭了电脑，第二天你后悔了，想恢复到最新的版本了怎么办)
8.git reflog (在git中，git会记录每一次命令)
(在你提交前你发现有个代码出错了，那么你可以删除它，或者使用)
9.git checkout -- text.html （修改后还没有暂存，你执行语句后会返回到最新的一次暂存，就是让这个文件回到最近一次git commit或git add时的状态
(当你想删除文件的时候 )可以直接rm 文件名
10。git rm text.html （从版本库中删除文件）
（一种情况是删错了，因为版本库里还有呢，所以可以很轻松地把误删的文件恢复到最新版本：）
11.git checkout -- text.html (因为上面每一次个操作都是一个修改)
（我们默认了你已经配演了ssh密钥）
12.git remote add origin git@github....ssh密钥 (连接远程仓库)
13.git push -u origin master (把分支为master的推送到远程)
第一次推送时 加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。
14.git push origin master
15.git checkout -b dev (-b相当于) //创建并切换到dev分支 git branch dev git checkout dev 16.git branch （查看当前分支） （随便在dev上写上什么东西） 17。git merge dev （当前是master，把dev的分支结果合并到master上） （当合并完成后，我么你就可以删除dev分支了） 18。git branch -d dev 我们注意到切换分支使用git checkout ，而前面讲过的撤销修改则是git checkout -- ，同一个命令，有两种作用，确实有点令人迷惑。 19.git switch -c dev （创建并且切换到dev分支） 20. git switch master （直接切换到已有的master分支） (当git无法自动合并分支时，必须要先解决冲突，解决冲突后、再提交，自动合并完成) （当你在开发dev分支的时候，在主分支上又一个bug分配给你了，你需要我dev分支储存起来） 21.git stash (储存当前分支) （然后切换到主分支，签出一个解决bug的分之来，然后修改完毕后，commit提交，切换到主分支然后合并分之） 22.git stash list (查看缓存区列表) 23.git stash apply (用于恢复，但是恢复之后缓存里面的内容并没有删除)，git stash drop 24.git stash pop 25.git stash apply stash@{0} //可以使用它恢复指定的stash (既然master分支上有一个bug，那么可想而知dev上也是有这么一个bug的，我们可以使用cherry-pick) 26.git cherry-pick 4c805e2 (能复制一个特定的提交、到当前分支) 27.git remote -v. (查看详细的远程) 28.git push origin master(推送到主分支、推送到其他分支) 标签管理 29.直接 git tag v0.1 (就可以打上标签) 30. git tag （查看标签） 31.git log --pretty=oneline --abbrev-commit （给历史的提交打上标签） $ git tag v0.9 f52c633 32.git show v0.9 （查看详细） 33.git tag -a v0.1 -m "version 0.1 released" 1094adb （可以指定内容信息）
